How to trigger a Flow based on the regarding object's type

2021-06-03

A quick post today about an annoyance I found while implementing a Flow for a customer.

One of my customers' requirements was that a Flow should be only started when a phone call is created on the Timeline of leads. For other entities, this Flow shouldn’t even be started.

Easy, right? Just add a row filter to check if field regardingobjectid_type equals ‘leads’. Wrong! To my surprise, while the Flow execution history shows that regardingobjectid_type contains the value ‘leads’, the row filter does not work.

So, what’s the solution? Add a trigger condition: @equals(triggerBody()?['_regardingobjectid_type'],'leads')
